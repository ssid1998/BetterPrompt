## 🎯 Sprint Goal

### What are we building?
We are making the application presentation-ready. This means allowing users to upload entire files (PDFs, TXT, Markdown) containing single or multiple prompts used for a single project (like a Data Maturity Assessment Tool). We will then take the raw JSON assessment from our local AI and display it in a beautiful, easy-to-read dashboard featuring scores, rule violations, pros/cons, and a rewritten "Perfect" prompt.

### Expected Outcome (What you will see)
By the end of this sprint, you will be able to:
1. Upload a `.pdf`, `.txt`, or `.md` file directly into the application.
2. Have the system automatically extract the text (even if it contains multiple spaced-out prompts).
3. See a polished UI dashboard displaying the overall score.
4. Read a clear list of Pros and Cons based on the prompt(s) submitted.
5. See a "Rewritten Perfect Prompt" generated by the AI to show the user how it *should* be done.

### Acceptance Criteria
- [x] User can drag-and-drop or select a PDF, TXT, or MD file.
- [x] Backend securely parses the text from these files (`pdf-parse` for PDFs).
- [x] The LLM System Prompt is updated to handle grading *multiple* prompts at once for a single goal.
- [x] The raw JSON response is replaced by a polished React component showing Score, Violations, and Rewrites.

## 📝 Tasks

### Task 1: Document Upload UI
**Description:** Create a file upload dropzone in the frontend.
**Technical Approach:** Add an `<input type="file" accept=".pdf,.txt,.md" />` component. Allow the user to toggle between pasting text or uploading a file.
**Status:** [x] Done

### Task 2: Backend File Parsing
**Description:** Extract raw text from uploaded files so the AI can read them.
**Technical Approach:** Install `pdf-parse`. Create an API utility that receives FormData, checks the file type, and extracts the text string.
**Status:** [x] Done

### Task 3: Multi-Prompt LLM Logic Update
**Description:** Ensure the AI knows it might be grading a *series* of prompts.
**Technical Approach:** Update the `systemPrompt` in `/api/assess/route.ts` to instruct the AI to evaluate the *entire collection* of prompts as a holistic strategy, and provide a comprehensive rewrite.
**Status:** [x] Done

### Task 4: The Feedback Dashboard UI
**Description:** Turn the raw JSON into a beautiful report card.
**Technical Approach:** Build React components for a Score indicator (progress bar), a Pros/Cons list, and a "Rewritten Prompt" text box with a copy-to-clipboard button.
**Status:** [x] Done
